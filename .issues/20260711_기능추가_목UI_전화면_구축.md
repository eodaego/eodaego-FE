## 제목

⚙️ [기능추가][목UI] 와이어프레임 기반 전 화면 목 UI 구축

## 본문

## 📝 현재 문제점

- 디자인 토큰(색상·타이포·스페이싱·래디우스)은 정의되어 있으나 실제 화면에 적용해 검증한 적이 없음
- 로그인 이후 화면이 임시 홈 스텁 하나뿐이라 앱의 핵심 플로우(코스 추천, 도감 수집)를 확인할 수 없음

## 🛠️ 해결 방안 / 제안 기능

- 와이어프레임 11장을 기준으로 전 화면 목 UI를 구축하여 디자인 토큰을 실제 화면에서 검증
- 하단 4탭(홈·지도·도감·즐겨찾기) + 중앙 카메라 버튼 구조의 탭 네비게이션 도입
- 백엔드 연동 없이 하드코딩 목 데이터로 화면과 화면 간 이동만 완성 (API 연동은 후속 작업)

## ⚙️ 작업 내용

- 목 데이터 및 신규 화면 라우트 구성
- 4탭 셸 + 카메라 진입 구조
- 화면 12종: 홈 / 코스 추천 위저드 / 추천 코스 결과 / 지도 / 촬영 / 퀴즈 / 정답 축하 / 도감 / 도감 상세 / 즐겨찾기 / 내 정보 / 탭 셸
- 로직이 있는 3곳(도감 필터·검색, 퀴즈 오답 잠금, 위저드 진행) 위젯 테스트

- 구현 계획: docs/superpowers/plans/2026-07-11-mock-ui.md
