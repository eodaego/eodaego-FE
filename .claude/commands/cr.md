# CR — Commit + Report (병렬)

`/commit`과 `/report`를 **병렬로** 실행하는 통합 커맨드. 두 작업은 독립적이라 동시에 돌려도 안전하다:

- 보고서는 **읽기 전용** + 결과를 `.report/`(gitignore, 추적 안 됨)에 기록
- 커밋은 **소스 파일만 명시적으로 `git add`** (보고서 파일은 손대지 않음)

## 실행 절차

### 1단계: 변경 스냅샷 1회 캡처

`git status` **한 번만** 실행해 변경 파일 목록과 현재 브랜치를 확보한다. 이 목록을 두 작업의 공통 입력으로 쓴다. (이후 두 작업은 `git diff`에 의존하지 않고 파일을 직접 읽는다 → 커밋이 먼저 끝나도 보고서 분석에 영향 없음)

### 2단계: 두 서브에이전트를 한 메시지에서 동시 dispatch (병렬)

- **에이전트 A — 커밋**: `.claude/commands/commit.md`(Smart Commit) 규칙을 그대로 따른다.
  - 기능 단위 분리 커밋, 한글 메시지, `<type> : <설명> #이슈`(브랜치명에서 `#숫자` 추출)
  - **push 금지 · Co-Authored-By 금지 · GPG 서명 금지 · 시크릿 파일 제외 · `git add .`/`-A` 금지(명시 지정)**
- **에이전트 B — 보고서**: `.claude/commands/report.md`(Report Mode) 규칙을 그대로 따른다.
  - `.report/[YYYYMMDD]_[#이슈]_[설명].md` 생성
  - **작성자/작성일/AI 언급 금지 · 민감정보 마스킹**
  - 변경 파일을 직접 읽어 분석(커밋 전후 무관)

### 3단계: 통합 보고

두 결과를 합쳐 출력한다:

- 커밋 표 (해시 · 메시지 · 파일 수)
- 보고서 저장 경로

## 주의

- 확인 없이 즉시 실행, push 안 함.
- 변경사항이 없으면 "커밋/보고할 변경사항이 없어요"로 종료.
