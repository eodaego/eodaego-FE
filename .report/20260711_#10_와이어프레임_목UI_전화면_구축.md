# 와이어프레임 기반 전 화면 목 UI 구축 보고서

### 📌 작업 개요

와이어프레임 11장을 기반으로 어대GO 전 화면 목 UI 구축. 디자인 토큰(AppColors/AppTextStyles/AppSpacing/AppRadius)을 실제 화면에 적용해 검증하고, 4탭 네비게이션·게스트 모드·지도 중심의 코스 추천 UX까지 완성. 백엔드 연동 없이 하드코딩 목 데이터로 화면과 플로우만 구현 (API 연동은 후속 작업).

**보고서 파일**: `.report/20260711_#10_와이어프레임_목UI_전화면_구축.md`
**이슈**: #10 · 브랜치 `20260710_#10_와이어프레임_기반_전_화면_목_UI_구축` (main 대비 33커밋)

### 🎯 구현 목표

- 홈 / 지도 / 도감(+상세) / 즐겨찾기 / 촬영 / 퀴즈 / 정답 축하 / 코스 추천 / 내 정보 — 전 화면 목 UI
- 4탭 셸(홈·지도·도감·즐겨찾기) + 중앙 카메라 FAB 네비게이션
- 로그인 없이 UI를 돌아볼 수 있는 개발용 게스트 모드
- API 연동 시 화면 코드 수정을 최소화하는 구조 (목 데이터 분리 + 환경 변수 분기 준비)

### ✅ 구현 내용

#### 1. 네비게이션 골격 — 4탭 셸 + 카메라 FAB

- **파일**: `lib/core/widgets/main_tab_shell.dart`, `lib/router/app_router.dart`, `lib/router/route_paths.dart`
- **변경 내용**: `StatefulShellRoute.indexedStack` 4브랜치(홈/지도/도감/즐겨찾기) + 중앙 카메라 FAB(64, canvas 링, 브랜드색 그림자). 촬영·퀴즈·축하·코스 추천·도감 상세·내 정보는 탭바 없는 루트 push
- **이유**: 기존 auth redirect 로직과 무수정 호환되는 구조. 탭 상태는 IndexedStack으로 보존

#### 2. 목 데이터 계층 (`lib/core/mock/`)

- **파일**: `mock_dogam.dart`, `mock_course.dart`, `mock_park_status.dart`
- **변경 내용**: 도감 80종(수집 24 — 수기 12종 + 자동 생성), 코스 6개(`durationMinutes` 필터 매칭 필드 포함), 공원 상태(날씨·혼잡도). `DogamCategory` enum이 카테고리 색 페어링(tint/dark)의 단일 소스
- **이유**: "24/80" 뱃지·수집률 30% 등 파생 수치가 리스트에서 계산되어 화면 간 불일치가 원천 차단. API 연동 시 폴더째 삭제 (단 `DogamCategory`는 core/constants로 이동 — 주석으로 표식)

#### 3. 화면 12종

- **홈**: 날씨·혼잡도 바 → 오늘의 추천 코스 프리뷰(reward CTA) → 도감 진행률 → 공식 사이트 링크
- **지도**: 약도 placeholder + 선택 코스 마커 + 드래그 시트(22%↔50%) — 시트는 "지금 보는 코스" + "코스 추천 받기" 진입 버튼
- **코스 추천**(`/course/recommend`): PageView 4스텝 + SmoothPageIndicator — 출입문/체류시간/관심분야(선택 시 자동 진행, 재탭 해제, 건너뛰기 = 전체) → 결과 CourseCard 가로 스와이프. "이 코스로 갈래!" → 지도 이동
- **도감**: 카테고리 필터 + 이름 검색(실동작) + 3열 그리드(수집/미수집) → 상세(어린이 눈높이 설명 카드)
- **촬영**: 다크 목 배경 + 점선 가이드 프레임 + 셔터 → 퀴즈. 카테고리 토글은 자동 인식 기획에 따라 제거
- **퀴즈 → 정답 축하**: 3지선다(오답 칸만 잠금, 해설 없음) → 노랑 전면 축하(앱 유일) + 도감 카드 scale-in
- **즐겨찾기**: 저장 코스 리스트 + 빈 상태. 정렬 실동작 — 전체/최근 저장순/소요시간순, 같은 칩 재탭 시 ↓↔↑ 방향 토글
- **내 정보**: 수집 통계 + 로그아웃(실동작, 기존 홈 스텁에서 이관) + 탈퇴 안내(Hard Delete 경고 subtitle)

#### 4. 선택 코스 공유 상태

- **파일**: `lib/core/providers/selected_course_provider.dart`
- **변경 내용**: `StateProvider<MockCourse>` — 홈 프리뷰·지도 마커·시트 헤더가 공동 구독, 갱신은 추천 페이지의 "이 코스로 갈래!" 한 곳
- **이유**: 화면마다 로컬 상태로 두면 "추천에서 골랐는데 지도는 딴 코스" 불일치 발생. 단일 소스로 3화면 동기화

#### 5. 게스트 모드 (개발용)

- **파일**: `lib/core/providers/guest_mode_provider.dart`, `lib/core/widgets/dialogs/login_gate_dialog.dart`, 로그인/촬영/즐겨찾기/내정보/시트/라우터
- **변경 내용**: 로그인 화면 "게스트로 둘러보기"(**kDebugMode 한정 노출**) → redirect 통과. 실효 제한은 `guestRestrictedProvider`(= 게스트 && !목모드)로 판단 — **목 모드(USE_MOCK_DATA=true)에서는 게이트 전부 해제**되어 전 기능 탐색 가능, 실 데이터 모드에서만 촬영 셔터(다이얼로그)·코스 추천(시트 인라인 안내) 게이트 동작
- **이유**: UI 개발 시 로그인 없이 돌아다니기 위한 도구. 릴리즈 빌드는 진입점 자체가 없어 "로그인 필수" 정책 유지

#### 6. 환경 변수 분기 준비

- **파일**: `lib/core/config/env_config.dart`, `.env.example`
- **변경 내용**: `USE_MOCK_DATA` 추가 — `EnvConfig.useMockData` getter, 미설정 시 false(실 데이터)
- **이유**: API 연동 시 Repository provider에서 목/실 구현체를 이 플래그로 분기. CI(`ENV_FILE` 시크릿)에 변수가 없어도 안전 기본값. 파이프라인 yaml 수정 불필요 (시크릿 내용 = .env 그 자체)

#### 7. 공용 위젯 신설 (`lib/core/widgets/`)

`AppBadge`(tint/dark 페어링 강제), `CategoryChip`, `DogamCard`, `CourseCard`, `AppBackAppBar`, `PreviousButton`(SVG 아이콘 + 햅틱 — 모든 뒤로가기 통일), `DashedRRectPainter`, `AppButton.reward` 팩토리. 지도 feature 내 `MapMarker`, `CourseSheet`

### 🔧 주요 변경사항 상세

#### 코스 추천 UX 변천

위저드(3스텝 화면) → 지도 시트 통합(칩 필터) → **최종: 전용 페이지(PageView 스텝) + 시트 축소**. 시트에 선택 코스와 추천이 공존하면 드래그 거리가 길어지는 실기 피드백을 반영해 분리. 스텝 자동 진행 타이머는 뒤로가기와의 경합으로 소프트락되는 레이스가 리뷰에서 발견되어 예약 시점 페이지 가드로 수정

#### 접근성·디자인 시스템 보정 (리뷰 발견)

- 칩·pill 3곳 터치 타깃 48 하한 미달 → `ConstrainedBox(minHeight: 48)` 보장
- 선택 카드 텍스트가 tint 배경 위에서 ink로 남던 것 → 카테고리 dark 페어링 적용
- 촬영 닫기(X) 버튼이 셔터 Stack 크기에 갇혀 겹쳐 보이지 않던 레이아웃 버그 수정
- 노랑(reward)은 홈 CTA + 축하 화면 정확히 2곳 유지, 혼잡도 뱃지는 노랑 금지 준수

**특이사항**:
- Firebase ID 토큰을 디버그 빌드에서 `developer.log`로 출력하는 개발 편의 코드 포함 (`firebase_auth_datasource.dart`, `kDebugMode` 가드) — 백엔드 API 테스트용. 릴리즈 빌드에는 미포함이나 디버그 로그 공유 시 토큰 노출 주의

### 📦 의존성 변경

- 신규 패키지 없음 (`smooth_page_indicator`, `flutter_svg`, `vibration` 등 기존 선언분만 사용)
- 테스트/목 단계 규칙: 신규 Riverpod codegen·freezed 미사용 (StateProvider 2개만), 화면 상태는 StatefulWidget + setState
